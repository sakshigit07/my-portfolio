Node.js Application Architecture

Deploying a robust Node.js environment on AWS EC2, leveraging Nginx as a reverse proxy and PM2 for advanced process management.

Server Environment

AWS EC2 (Linux) Node.js & NPM PM2 Process Manager Nginx Reverse Proxy Git CI/CD

Why this setup?

Running Node.js behind Nginx improves performance, security, and allows the application to be served through standard Port 80 (HTTP). This abstracts the internal port 3000 away from the public, providing a professional entry point.

Reverse Proxy: Nginx handles Port 80 → Port 3000 forwarding.
Process Persistence: PM2 ensures the app restarts on crashes/reboots.
Hardened Security: Restricted inbound rules to keep origin safe.

🛠️ Deployment Workflow & Proxy Logic

*Nginx Reverse Proxy: Handling external HTTP traffic and routing to internal Node.js runtime*

Server Hardening & Configuration:

1. Provisioning: Launched an Amazon Linux instance and initialized the environment with Node.js, NPM, and Git.

sudo yum install nodejs npm git -y
git clone https://github.com/sakshigit07/node-app.git

2. Application Launch: Installed dependencies and configured PM2 to manage the background process for 24/7 uptime.

npm install
sudo npm install -g pm2
pm2 start app.js

3. Nginx Reverse Proxy: Configured the server block to redirect traffic from the web port (80) to the application core (3000).

location / {
  proxy_pass http://localhost:3000;
}
