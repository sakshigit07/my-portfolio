Automated Portfolio Hosting

High-performance static hosting using Amazon S3 and CloudFront with a fully automated GitOps deployment pipeline via GitHub Actions.

Deployment Stack

Amazon S3 GitHub Actions

Project Objective

To build a modern, zero-maintenance hosting solution that allows developers to deploy code simply by pushing to a repository. This project focuses on minimizing latency via global CDN edges while ensuring the S3 origin remains strictly private.

Continuous Deployment: Changes are live within seconds of a `git push`.
Origin Access Control: Ensuring S3 buckets aren't public-facing.
Global Edge: Reducing TTFB (Time to First Byte) via CloudFront.

🛠️ CI/CD Workflow & Architecture

Automated Execution:

1. Trigger: The pipeline starts when a commit is pushed to the `main` branch. GitHub Actions initiates the workflow using pre-configured AWS credentials (via Secrets).

2. Build & Test: The environment is prepared, and files are optimized/minified for production deployment.

3. Sync to S3: The action uses the AWS CLI to sync only changed files to the **Amazon S3** bucket, keeping the hosting storage clean and efficient.

4. Invalidation: Finally, the pipeline creates a **CloudFront Invalidation** (`/*`) to clear the edge cache, ensuring users see the updated content immediately.

Key Outcomes

  • Automated SSL via ACM
  • GitOps-driven Infrastructure
  • Reduced Hosting Costs (Serverless)
  • AWS OAC Security Compliance
