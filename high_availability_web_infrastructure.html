Scalable Infrastructure on AWS

A resilient, three-tier architecture designed to eliminate single points of failure and scale automatically based on traffic demand.

Provisioned Infrastructure

VPC & Networking EC2 & ASG ALB Multi-AZ RDS Amazon Linux IAM & Security Groups

Project Objective

The goal was to architect a production-grade environment that moves beyond single-server setups. By leveraging multiple Availability Zones (AZs) and managed services, the infrastructure ensures business continuity even during data center outages.

Multi-AZ Redundancy: Resources are distributed across multiple Availability Zones to ensure high availability.
Elastic Scalability: An Auto Scaling Group (ASG) monitors CPU metrics to launch or terminate EC2 instances dynamically.
Data Resilience: Amazon RDS is deployed in a Multi-AZ configuration with a synchronous standby replica for instant failover.
Traffic Distribution: An Application Load Balancer (ALB) acts as the single entry point, health-checking instances and routing traffic efficiently.

☁️ Architecture Deep Dive

*Resilient Three-Tier VPC Architecture with ALB and Multi-AZ RDS*

The Workflow Execution:

1. Networking: A custom VPC (10.0.0.0/16) provides an isolated network environment where traffic enters through the Internet Gateway (IGW).

2. The Entry Point: The Application Load Balancer (ALB) sits in the Public Subnets, health-checking instances and distributing HTTP/S traffic efficiently.

3. The Compute Layer: EC2 instances operate within an Auto Scaling Group. If an instance fails, the ASG automatically triggers a self-healing replacement.

4. The Data Layer: RDS is isolated in the deepest Private Subnets, accessible only from the App tier Security Group following the principle of least-privilege.
