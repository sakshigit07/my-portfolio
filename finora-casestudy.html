Building Finora: Personal Finance on AWS

A full-stack web application designed for organized income and expense tracking, implemented with foundational cloud services.

Tech Stack & Infrastructure

AWS EC2 (Amazon Linux) DynamoDB (NoSQL) PHP 8.x Backend Nginx Web Server AWS SES IAM Security HTML, CSS, JavaScript

Project Objective

Finora is a personal finance management web application designed to help users track, manage, and analyze their income and expenses in an organized and secure way. It provides a simple dashboard where users can record daily transactions, categorize spending, and gain insights into their financial habits. The main goal of Finora is to make money management easy, visual, and accessible, especially for individuals who want a lightweight and clean alternative to complex finance apps.

Secure Authentication: User login/registration handled via server-side PHP logic and session management.
Transaction Logging: Users can securely record and categorize income and expenditure details.
DynamoDB Integration: Transaction data is modeled and persisted using DynamoDB for fast reads/writes.
Dashboard Summaries: Basic reporting/visualization logic to show spending trends and net balance.

☁️ Architecture & Backend Workflow on AWS

*Architecture showing User Nginx, PHP and DynamoDB flow, all on a single EC2 instance.*

Detailed Request Flow:

The workflow starts from the user’s browser, where a user initiates actions such as signup or login. These requests are sent to an Amazon EC2 instance, which acts as the core compute layer of the application.

On the EC2 instance, Nginx is configured as the web server. Nginx handles incoming HTTP requests and works as a reverse proxy, forwarding dynamic requests to PHP-FPM. PHP-FPM processes the application’s backend logic, such as validating user input, managing sessions, and handling authentication logic for Finora.

For data storage and session-related operations, the backend interacts with Amazon DynamoDB. DynamoDB is used to read and write user and session data in a scalable, serverless manner. This removes the need for managing traditional database servers and helps handle variable workloads efficiently.

To support authentication-related communication, such as sending verification or system emails, the workflow integrates Amazon SES (Simple Email Service). PHP-FPM sends requests to SES, which then handles reliable email delivery.

Security and permissions are managed using AWS IAM roles. Instead of hardcoding credentials, the EC2 instance uses an IAM role with appropriate permissions to securely access DynamoDB and SES. This approach follows AWS best practices for security and access control.

Overall, this architecture demonstrates how Finora can be structured using AWS-native services to build a secure, scalable, and modular backend workflow, while keeping the application logic centralized on EC2.

Due to working within an AWS sandbox / learner account, this architecture could not be fully deployed for public or multi-user access. However, the complete design, configuration, and request flow were implemented and validated within the sandbox environment, providing hands-on experience with real AWS services and production-style architecture.
